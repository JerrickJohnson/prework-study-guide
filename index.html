HTML

  • Hypertext Markup Language (HTML).
  • Defines the basic framework for the page and the elements we want to appear on it.
  • HTML does not determine the styling, specific layout, or make the page interactive.
  • Defines Text Elements on the page, images on the page, the order in which the elements appear on the page and which text elements are the primary and secondary headings.
  • Elements are part of a webpage that holds content or performs a specific task.
  • HTML Elements are usually made up of opening tag and a closing tag.
  • All HTML documents are composed of an html element that contains a head and body element.
  • The head element contains information about the webpage.
  • Included in the head element are the title, style, link, meta, script, and noscript elements.
  • The body element represents the visible content shown to the user.
  • Inside the body element h1, h2, h3, h4, h5 and h6 elements represent the level of the heading.
  • Other elements included in the body element are p, strong, em, br, img, a, ul, ol, and li.
  • There are also Semantic Elements that are often used to divide a page.
  • Semantic Elements include article, aside, details, figcaption, figure, footer, header, main, nav, and section elements.
  • HTML attributes are used to provide additonal information about an element.
  • Global attributes are attributes that can be used on any element, although they might not have an impact on or change some elements.
  • Common global attributes are data-*, dir, lang, style, class, and id.
  • Non-Global Attributes are other elements that can only be used with specific elements.
  • The common element-specific attributes include alt, href, placeholder, src, and value.

CSS

  • Cascading Style Sheets (CSS)
  • A stylesheet language used to describe the presentation of a document written in HTML or XML
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline CSS is used when a you want to style an element directly in the HTML file.
  • Inline Styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Internal CSS style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • The Internal style sheet rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code.
  • External CSS Style sheets are the most common, it allows you do keep all CSS rules in a separate file, which makes design changes easier.
  • When using an external style sheet you will use a link element placed in the HTML's head.
  • Rules or Declarations contain two important commponents: the CSS property we want to apply and the value of the property.
  • An example the declaration color:red; includes CSS property color and the value for that property, red.
  • Other CSS rule examples are margin and padding.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The CSS box model- every HTML element is essentially a box. Each box consists of margins outside the element, and the element's borders, padding, and content.
  • CSS determines the styles of a webpage in order to make it visually appealing and can also make a website simpler to use with a polished design.

Git

  • Git is the free and open source distibuted version control system that's responsible for everything GitHub related that happens locally on your computer.
  • git status: Checks what branch we are currently on
  • git checkout -b branch-name: Creates a new branch and switches to it
  • git add -A: The add command adds modifications in the current working branch to the staging area and the -A flag indicates that we want to add all changes
  • git commit -m "describe what commit contains": Commits the staged files to be commited with a commit messge that describes what the commit contains
  • git pull origin main: Uses the pull command to receive a branches modifications into the local environment. The origin keyword indicates the source of the pull will be in the GitHub repo. The main keyword indicates the branch
  • git push origin feature/starter-code: In this example we are using the command to push the changes we have locally to our remote GitHub branch

JavaScript

  • JavaScript is a powerful language that can add interactivity to a website.
  • Using JavaScript you can power your websites with effects, interactivity, data communication, and much more.
  • A variable is a named container that allows us to store data in our code.
  • You start by declaring a variable with the let keyword, followed by the name you give the variable.
  • A semicolon at the end of a line indicates where a statement ends.
  • JavaScript is case sensitive meaning myVariable is not the same as myvariable.
  • After declaring a variable, you can give it a value: myVariable = "Bob";
  • You retrieve the value by calling the variable name: myVariable;
  • After assigning a value to a variable, you can change it later in the code: let myVariable = "Bob"; myVariable = "Steve";
  • You can do both declaring a variable and give it a value on the same line: let myVariable = "Bob";
  • Variables may hold values that have different data types: String(sequence of text), Number(numbers), Boolean(true/false value), Array(structure that allows you to store multiple values in single reference), Object(can be anything)
  • You can make comments of text that can be be added along with code and the browser ignores the text marked as comments using /*and then closing with */. With no line breaks you can use // in front of your comment.
  • An operator is a mathematical sybol that produces a result based on two values(or variables).
  • Some simple operators used in JavaScript are Addition(+), Subtraction(-), Multiplication(*), Division(/), Assignment(=), Strict Equality(===), Not(!), Does Not Equal(!==)
  • Conditionals are code structures used to test if an expression returns true or not.
  • A very common form of conditionals is the if...else statement.
  • Functions are a way of packaging functionality that you wish to reuse.
  • Events are code structures that listen for activity in the browser and run code in response.
  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page.
  • Control flow can be interrupted. We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
